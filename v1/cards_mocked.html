Para probar los posibles resultados de un pago, utiliza alguno de los siguientes prefijos en el campo name de /card_tokens o en el campo cardholderName:


APRO: Pago aprobado

CONT: Pago pendiente

CALL: Rechazo llamar para autorizar

FUND: Rechazo por monto insuficiente

SECU: Rechazo por código de seguridad

EXPI: Rechazo por fecha de expiración

FORM: Rechazo por error en formulario

OTHE: Rechazo general

Arma tu propio checkout usando nuestra UI

La SDK de MercadoPago te provee la interfaz gráfica resuelta para cada uno de los pasos propuestos para completar un pago. Si querés tener control sobre el flujo de compra de tu usuario, podes armar el flujo que quieras, sin preocuparte por la UI del mismo.

Usá los servicios de MercadoPago para armar tu checkout

También encapsulamos los servicios en una clase para que puedas consumirlos y proveerle la interfaz gráfica que desees a tu usuario sin necesidad de entender cómo conectarte con nuestras APIs.
