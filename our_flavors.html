Soluciones propuestas

Ofrecemos tres formas distintas para que implementes nuestra SDK.

Checkout completo( Complete checkout flow tool): agregando pocas lineas de código y las configuraciones que desees, vas a tener en tu aplicación todo el flujo de pagos propuesto por MercadoPago. Vos ingresas lo que queres cobrar y nosotros te damos como respuesta un pago, ya sea aprobado, o no.

Interfaz gráfica para recolección de datos (Data collection UI): Es el mismo escenario que el anterior, pero el pago lo realizás vos desde tu servidor. Incluso puedes modificar el flujo a tu gusto. En este caso ingresas lo que quieres cobrar y luego vamos a retornarte los datos necesarios para hacer el pago: medio de pago (payment_method) y en el caso de que sea una tarjeta, su token (card_token) junto al banco y cantidad de cuotas seleccionadas (issuer, installment).

Checkout personalizado (Customizable UI): En este caso nuestra SDK te resuelve la conexión con nuestas APIs para que vos armes el flujo de pago a tu gusto, sin usar nuestra interfaz gráfica propuesta.

A continuación hacemos una comparación sobre nuestras soluciones:

Checkout completo
  • Implementación inmediata.
  • Flujo no customizable.
  • Interfaz gráfica no customizable.
  • Sólo se obtiene la información requerida para el checkout.
Interfaz gráfica para recolección de datos
  • Debes manejar la información del chekcout y realizar los pagos contra nuestra API.
  • Flujo customizable.
  • Interfaz gráfica no customizable.
  • Sólo se obtiene la información requerida para el checkout.
Checkout personalizado
  • Debes manejar la información del chekcout y realizar los pagos contra nuestra API e implementa la interfaz gráfica de su flujo.
  • Flujo customizable.
  • Interfaz gráfica customizable.
  • Obtienes la información que desees, además de la requerida por el Checkout.
