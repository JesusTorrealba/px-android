PaymentVault

El más completo de nuestros componentes visuales es PaymentVault. Consiste en un flujo que al ser finalizado retorna todos los datos que seleccionó el usuario. Es decir, utilizando este componente, estás en condiciones de enviar los datos a tu servidor pues tendrás a disposición el medio de pago seleccionado (PaymentMethod) y si el usuario paga con tarjeta, el banco (Issuer), las cuotas (PayerCost) y la información de la tarjeta encriptada (Token).
Para utilizarlo, agregá un botón:

Luego, cuando el botón es presionado, desde nuestra SDK invocá a PaymentVault y esperá el resultado para conocer los datos que seleccionó el usuario dentro del flujo. 

                        
// Método ejecutado al hacer clic en el botón
public void submit(View view) {
  // Iniciá el flujo de recolección de datos
  new MercadoPago.StartActivityBuilder()
    .setActivity(activity)
    .setPublicKey("TEST-ad365c37-8012-4014-84f5-6c895b3f8e0a")
    .setAmount(BigDecimal.valueOf("100.0"))
    .startPaymentVaultActivity();   
}

@Override
protected void onActivityResult(int requestCode, int resultCode, 
Intent data) {

    if(requestCode == MercadoPago.PAYMENT_VAULT_REQUEST_CODE) {
      if(resultCode == RESULT_OK) {
         PaymentMethod selectedPaymentMethod = 
          (PaymentMethod) data.getSerializableExtra("paymentMethod");
         Issuer selectedIssuer = 
          (Issuer) data.getSerializableExtra("issuer");
         PayerCost selectedPayerCost = 
          (PayerCost) data.getSerializableExtra("payerCost");
         Token createdToken = 
          (Token) data.getSerializableExtra("token");
      
        // Enviá estos datos a tu servidor para finalizar el pago.
            
        } else {
            // Ups, ocurrió un error.
        }
    }
}

Cobra con tarjeta

Te proveemos una llamada a nuestro componente que te resuelve la recolección de los datos de tarjeta de tu usuario de una forma intuitiva y segura. Sólo tienes que invocar el formulario de tarjeta y esperar el resultado: PaymentMethod (marca de la tarjeta), Issuer (banco de la tarjeta) y Token (información encriptada de la tarjeta). Sigue navegando si quieres ver cómo cobrar en cuotas..

¿Cómo adivinamos la marca de la tarjeta?

Dado el BIN de la tarjeta (6 primeros dígitos), podemos determinar el medio de pago (Visa, Mastercard, Amex) y mediante expresiones regulares, muchas veces podemos determinar el banco.

Cobra con tarjeta - Con cuotas

Si deseas cobrar en cuotas, ahora no debes llamar al formulario de tarjeta, sino a un flujo que se encarga de la recolección de datos de la tarjeta y de la selección de cuotas. A los parámetros de respuesta del paso anterior, además debes escuchar el PayerCost (las cuotas seleccionadas por el usuario).

Cobra con tarjeta - Bancos

Para recordarlo, llamamos BIN a los seis primeros dígitos de la tarjeta. La mayoría de las veces, estos nos ayudan a definir el banco para poder ofrecerle las promociones que tiene el usuario con su tarjeta, pero otras tantas, no podemos determinar el banco y debemos pedirle al usuario que nos lo indique.

Por eso, también te ofrecemos una pantalla de selección de bancos para que utilices en tu flujo. No te olvides de esperar la respuesta a esta pantalla. El usuario seleccionó un banco y nosotros te lo devolvemos!

Y si tampoco te convence la pantalla para bancos, podes armar tu propia interfaz gráfica para la selección de bancos. Usando el servicio que mostramos a continuación podes manipular la lista de bancos y mostrarlos como mejor te parezca.

Cobra con tarjeta - Cuotas

Para el banco seleccionado en el punto anterior, el usuario debe seleccionar un plan de cuotas. Aquí te ofrecemos una pantalla para mostrar las cuotas.

Pero recuerda que puedes armar tu propia interfaz gráfica!

Cobra con tarjeta - Promociones

Nuestro equipo comercial trabaja para ofrecer las mejores promociones a los usuarios. Por eso, para que puedas contarle a tus usuario esos beneficios, te invitamos a mostrarle las promociones.

Si no te gusta como mostramos las promos, podes armar tu propia interfaz gráfica! Cómo? Usando el servicio que mostramos a continuación, podes manipular la lista de promociones y mostrarlas como mejor te parezca.

Medios de pago

COMENTARIO INTERNO: Prueben de hacer clic en la pantalla de la derecha como si navegaran el checkout!

Uno de los parámetros que conforma el pago es el medio con el que se está ejecutando el mismo. Este puede ser la marca de la tarjeta o la entidad a la que el usuario debe ir a finalizar el pago en efectivo.
Basados en nuestro foco en el usuario y luego de haber realizado A/B Testings, los números nos indican que la mejor forma de mostrar los medios de pago disponibles es agrupándolos en base a la forma en que el usuario desea pagar. Esta forma de agrupar, nos permite darte la flexibilidad de filtrar los tipos y medios de pago para así armar un árbol que permita la rápida selección por parte del usuario.
Cómo verás más adelante, también tendrás la posibilidad de mostrar el listado completo usando nuestra UI o armar la tuya.

Medios de pago - Tarjetas

Cuando el usuario selecciona que quiere pagar con una tarjeta, éste debe indicarte si la misma es de crédito, de débito o prepaga. Estos son tipos de pago y no un medio en sí. Además, estos tipos dependen del país en el que estés operando.

* Consulta los tipos y medios de pago disponibles aquí.

Medios de pago - Efectivo

Si el usuario desea pagar en efectivo, le indicamos el listado de entidades en las que puede finalizar el pago. También le indicamos el tiempo de acreditación para que de acuerdo a su selección, puedas recibir el dinero más rápido y él, su producto. En nuestra próxima versión vamos a indicarle mediante geolocalización cuál es la entidad más cercana!

Medios de pago - Transferencia Bancaria

Es un caso muy similar al anterior. Lo que aquí sucede es que al hacer la transacción por transferencia bancaria, son distintas las entidades que ofrecen este servicio. En nuestra próxima versión vamos a indicarle mediante geolocalización cuál es la entidad más cercana!

Medios de pago - Sin agrupar

Si no es de tu agrado la forma en la que agurpamos los medios de pago, también te proponemos una pantalla que los lista sin criterio alguno. Lo único que tenés que indicarnos son los tipos de pago que soportas, de esa forma los filtramos y mostramos aquellos que cumplen con tus condiciones.

No te olvides de esperar la respuesta a esta pantalla. El usuario seleccionó un medio de pago y nosotros te lo devolvemos!

Y si tampoco te convence la pantalla sin agrupar, podes armar tu propia interfaz gráfica para medios de pago. Usando el servicio que mostramos a continuación podes manipular la lista de medios de pago y mostrarlos como mejor te parezca.

Resultados de pago

Una vez que se recolectaron todos los datos necesarios para efectuar el cobro, si utilizaste el Checkout Básico, el posteo lo hacemos nosotros. Pero si utilizaste alguna integración personalizada, debes efectuar el cobro desde tu servidor. En este link podes ver cómo enviarnos los datos del cobro para ver el dinero refleajado en tu cuenta de MercadoPago.


Nuestra SDK te ofrece una clase llamada MerchantServer que es un REST Client con métodos que te serán útiles a la hora de realizar ciertas acciones desde tu servidor, pero de rápida adaptación en los entornos nativos. Sólo tenes que decirnos cuál es la URL base de tu servidor (por ejemplo api.mercadopago.com/v1/), cual es el identificador del recurso (URI, por ejemplo /payment_methods) y una serie de parámetros que para cada caso en particular vamos a requerirte. Desde tu servidor, sólo retorna la misma respuesta que nuestros servicios responden.

Resultados de pago - Medios offline

Si el usuario eligió un medio de pago offline (efectivo, transferencia bancaria), debes indicarle qué pasos debe realizar para poder completar el pago, ya que el mismo se encuentra en estado pendiente.


Hace un tiempo le contábamos los pasos en un mail al usuario, pero ahora te damos la posibilidad de que vos mismo puedas contarle dentro de tu aplicación cómo puede hacer para concretar el pago y tener más rápido su producto.

Ofrecemos una pantalla de instrucciones que recibe un pago y obtiene los pasos a seguir para el medio de pago que seleccionó el usuario como puedes ver a tu derecha.
